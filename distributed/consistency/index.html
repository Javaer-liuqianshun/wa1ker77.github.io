# 主从不一致分析

如上图是最常见的数据库集群架构，一主多从，主从同步，读写分离。

（1）一个主库提供写服务

（2）多个从库提供读服务，可以增加从库提升读性能

（3）主从之间同步数据

数据库主从会出现不一致问题，因为主从同步有时延，这个时延期间读从库，可能读到不一致数据。

（1）服务发起一个写请求

（2）服务又发起一个读请求，此时同步未完成，读到旧数据(不一致的脏数据)

（3）数据库主从同步最后才完成

# 主从不一致解决方案

方案一：忽略

如果业务能够接受，别把系统架构搞的太复杂。

方案二：强制读主
