# Redis实现分布式锁

在学习秒杀系统时，会有超卖的情况的出现。使用Redis实现分布式锁来解决超卖，如果不是分布式系统，使用Java的synchronized和Lock即可。

# 场景

用apache benchmarking压力测试：

ab -n 100 -c 10 http://localhost:8080/sell/skill/order/123456

模拟100个请求，并发量为10。出现超卖的情形：

国庆活动,皮蛋粥特价,限量份100000 还剩: 99990份 该商品成功下单用户数目:100 人

剩余数 + 下单数 = 100090 > 10000，说明出现超卖。

分析：但并发量大的时候，都会在Thread.sleep(100)等待100ms，会到导致减库存操作虽然执行但是没有保存，在并发中的每一个 线程拿到的stockNum都是没有保存之前的stockNum，所以导致只下单，没有真正减库存操作，俗称超卖的现象。

# 解决方案

使用Redis实现分布式锁

@Component
@Slf4j
public class RedisLock {
    @Autowired
    private StringRedisTemplate redisTemplate;

    /**
     * 加锁
     *
     * @param key
     * @param value 当前时间 + 超时时间
     * @return
     */
    public boolean lock(String key, String value) {

        // 如果key对应的value缺省就设置锁
        if (redisTemplate.opsForValue().setIfAbsent(key, value)) {
            return true;
        }

        String currentValue = redisTemplate.opsForValue().get(key);
        // 如果锁过期
        if (!StringUtils.isEmpty(currentValue) && Long.parseLong(currentValue) < System.currentTimeMillis()) {
            // 获取上一个锁,并设置新的锁
            String oldValue = redisTemplate.opsForValue().getAndSet(key, value);
            if (!StringUtils.isEmpty(oldValue) && oldValue.equals(currentValue)) {
                log.info("[redis分布式锁]获取锁成功")
                return true;
            }
        }
        log.info("[redis分布式锁]获取锁失败")
        return false;
    }

    public void unlock(String key,String value){
        try{
            String currentValue = redisTemplate.opsForValue().get(key);
            if (!StringUtils.isEmpty(currentValue) && currentValue.equals(value)){
                redisTemplate.opsForValue().getOperations().delete(key);
            }
        }catch (Exception e){
            log.error("[redis分布式锁]解锁异常");
        }
    }
}
